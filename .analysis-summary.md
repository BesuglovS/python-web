# Анализ python-web: найденные проблемы и план улучшений

## P0 — Критические

1. **Порядок уроков нелогичен**
   - Множества (21) перед списками (22), кортежами (23), словарями (24)
   - itertools (29) перед «Модули и import» (30)
   - Нужно: Списки→Кортежи→Словари→Множества, Модули→itertools

2. **Песочница уязвима**
   - Строковый блоклист вместо AST-анализа
   - Выполнение через shell (bypass_shell=false)
   - Нет rate-limiting

## P1 — Важные

3. **Пропущенные темы Python**
   - f-строки, лямбды, генераторы/yield, декораторы, рекурсия
   - type hints, pip/venv, отладка

4. **Нет серверного сохранения прогресса**
   - Только localStorage

5. **Нет интерактивного REPL**

## P2 — Желательные

6. Нет визуальных схем/диаграмм
7. Нет чит-листов по разделам
8. Service Worker без stale-while-revalidate
9. Финальный проект без выбора уровней сложности
