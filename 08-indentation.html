8. Отступы (indentation)

Как Python использует отступы для определения блоков кода

Зачем нужны отступы?

В отличие от многих языков, Python не использует фигурные скобки {} или команды Begin end для выделения блоков. Вместо этого блок кода определяется отступами: 

if True:
    print("Этот код внутри if")
    print("Это тоже внутри if")
print("А это уже вне блока if")

Пример с if

age = 20
if age >= 18:
    print("Доступ разрешён")
    print("Добро пожаловать!")
else:
    print("Доступ запрещён")
    print("Вам нет 18 лет")

Пример с циклом for

for i in range(3):
    print(f"Итерация {i}")
    print("Всё ещё внутри цикла")
print("Цикл завершён")

Тело функции

def greet(name):
    print(f"Привет, {name}!")
    print("Рады вас видеть!")
greet("Анна")

Стандарт PEP 8

PEP 8 рекомендует использовать 4 пробела для каждого уровня отступа. Не смешивайте табуляцию и пробелы! 

# Правильно (4 пробела)
if x > 0:
    print("Положительное")

Автоформатирование в VS Code

Редактор VS Code умеет автоматически расставлять отступы при сохранении файла:

  • Установите расширение Python (от Microsoft).
  • Откройте настройки (Ctrl + ,) и включите Editor: Format On Save.
  • Выберите средство форматирования: Python › Formatting: Provider — рекомендуется autopep8 или black.
  • Теперь при сохранении файла (Ctrl + S) VS Code сам поправит отступы, лишние пробелы и приведёт код к стандарту PEP 8.

Можно также отформатировать код вручную — нажмите Shift + Alt + F (Windows) или Shift + Option + F (macOS).

Автоформатирование в PyCharm

PyCharm также предоставляет мощные средства для работы с отступами:

  • Настройте автоформатирование при сохранении: File → Settings → Tools → Actions on Save → включите Reformat code.
  • Для ручного форматирования используйте Ctrl + Alt + L (Windows/Linux) или Cmd + Option + L (macOS).
  • PyCharm может автоматически расставлять отступы по мере набора текста — эта опция включена по умолчанию.
  • Настроить стиль отступов можно в Settings → Editor → Code Style → Python: выберите размер отступа (рекомендуется 4), использование пробелов вместо табуляции и т.д.

Распространённые ошибки

  • IndentationError: unexpected indent — лишний отступ там, где он не нужен
  • IndentationError: expected an indented block — отсутствует отступ после if, for, def и т.д.
  • Смешивание табуляции и пробелов — приводит к трудноуловимым ошибкам 
# Ошибка: нет отступа после if
if True:
print("Ошибка!")
